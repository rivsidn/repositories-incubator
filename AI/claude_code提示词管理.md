### GNU Stow

● GNU Stow 是一个符号链接管理工具，主要用于管理软件包的安装和配置文件的组织。

  主要功能

  1. 符号链接农场管理器：Stow 通过创建符号链接来管理文件，而不是复制或移动文件
  2. 软件包管理：可以将软件安装在独立目录中，然后通过 Stow 创建链接到系统目录
  3. 配置文件管理：特别适合管理点文件（dotfiles），如 .bashrc、.vimrc 等

  工作原理

  Stow 将每个"包"存储在独立目录中，然后在目标目录创建指向这些文件的符号链接。例如：
  - 源目录：~/dotfiles/vim/.vimrc
  - 目标链接：~/.vimrc → ~/dotfiles/vim/.vimrc

  常用命令

  stow package_name    # 创建符号链接
  stow -D package_name # 删除符号链接
  stow -R package_name # 重新创建链接

> 分别给介绍一下这些工具

● 好的，我来详细介绍一下这些配置文件管理工具：

  1. GNU Stow

  符号链接农场管理器，最简单直接的选择

  工作原理：
  - 创建符号链接来管理配置文件
  - 把源目录的文件结构"折叠"到目标目录

  特点：
  - ✅ 极其简单，几乎零学习成本
  - ✅ 轻量级，Unix 哲学的典范
  - ✅ 修改源文件立即生效（因为是符号链接）
  - ❌ 功能单一，只管理符号链接
  - ❌ 没有模板、加密等高级功能
  - ❌ Windows 支持较差

  适用场景：
  - 简单的配置文件切换
  - 已有良好目录结构的项目
  - 喜欢简单工具的用户

  2. Chezmoi

  现代化的配置文件管理器，功能最全面

  工作原理：
  - 使用实际文件而非符号链接
  - 通过模板系统处理机器差异
  - 内置 Git 集成

  特点：
  - ✅ 功能丰富：模板、加密、密码管理器集成
  - ✅ 跨平台支持优秀（Linux/Mac/Windows）
  - ✅ 活跃的社区和完善的文档
  - ✅ 内置 Git 操作命令
  - ❌ 学习曲线较陡
  - ❌ 相对复杂，可能过度设计

  适用场景：
  - 需要在多台不同配置的机器间同步
  - 配置文件包含敏感信息
  - 需要条件化配置（如根据操作系统）

  3. rcm

  Thoughtbot 出品的配置文件管理工具

  工作原理：
  - 基于符号链接
  - 提供了一套完整的命令行工具

  特点：
  - ✅ 设计优雅，文档完善
  - ✅ 支持主机特定配置
  - ✅ 标签系统管理不同配置集
  - ❌ 主要面向 Unix 系统
  - ❌ 社区相对较小

  适用场景：
  - Ruby 开发者（Thoughtbot 风格）
  - 需要标签系统组织配置

  4. Dotbot

  基于 YAML 的配置文件引导工具

  工作原理：
  - 使用 YAML 配置文件定义安装过程
  - 一键安装所有配置

  特点：
  - ✅ 配置即代码，易于版本控制
  - ✅ 支持插件扩展
  - ✅ 适合自动化部署
  - ❌ 需要编写 YAML 配置
  - ❌ 主要关注初始化，日常管理功能较少

  适用场景：
  - 新机器的快速配置
  - 喜欢声明式配置的用户

  5. Homesick

  Ruby 实现的 dotfiles 管理器

  工作原理：
  - 基于 Git 和符号链接
  - 使用"城堡"概念管理不同配置集

  特点：
  - ✅ 简单直观的命令
  - ✅ 支持多个配置仓库
  - ❌ 需要 Ruby 环境
  - ❌ 开发不太活跃

  适用场景：
  - Ruby 开发者
  - 需要管理多个独立的配置集

  6. yadm (Yet Another Dotfiles Manager)

  基于 Git 的配置文件管理器

  工作原理：
  - 直接使用 Git，但避免了 $HOME 成为 Git 仓库
  - 支持加密敏感文件

  特点：
  - ✅ Git 原生命令，学习成本低
  - ✅ 支持加密和模板
  - ✅ Bootstrap 脚本支持
  - ❌ 功能不如 chezmoi 丰富

  适用场景：
  - Git 重度用户
  - 想要 Git 原生体验

  对于你的需求

  考虑到你要管理 CLAUDE.md 并与 GitHub 集成，我的推荐顺序是：

  1. Stow - 如果你喜欢简单直接
  2. Chezmoi - 如果你预期会有更复杂的需求
  3. yadm - 如果你想要 Git 原生体验
